# 工作疑问

## 父组件生命周期 updated 触发时机

**触发时机**：
- 父组件自身数据变化导致重新渲染
- 子组件 emit 事件更新了父组件的 data
- 父组件的 props 从父级传来并发生变化
- 监听器（watch）触发更新

**注意**：
- `updated` 在 DOM 更新**后**触发
- 如果只是子组件内部变化，不会触发父组件的 `updated`
- 需要子组件通过 `$emit` 通知父组件才会触发

**示例**：
```vue
// 父组件
<template>
  <div>
    <ChildComponent @update-data="handleDataUpdate" />
    <p>{{ message }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return { message: '' }
  },
  updated() {
    console.log('父组件 updated 触发')
  },
  methods: {
    handleDataUpdate(data) {
      this.message = data // 这会触发父组件 updated
    }
  }
}
</script>
```

## 分离组件设计思路

**问题**：分离组件设计思路在某些场景下不可行

**常见原因**：
1. **状态共享困难**：分离后状态管理复杂，需要大量 prop 传递和 event 通信
2. **性能问题**：组件边界划分不当导致不必要的重渲染
3. **过度拆分**：拆分粒度太细，增加了维护成本

**解决方案**：
- 重新评估组件边界，适当合并相关功能
- 使用状态管理库（Pinia/Vuex）统一管理共享状态
- 考虑使用 composition API 重构逻辑

**最佳实践**：
- 按功能模块拆分，而非按 UI 元素
- 保持组件高内聚低耦合
- 避免过深的组件嵌套

## 工程问题：TypeScript 版本不兼容

**问题**：`typescript@7.0.2` 和 `@typescript-eslint@5.62.0` 版本不兼容

**原因分析**：
- `@typescript-eslint` 5.x 版本要求 TypeScript 4.x 或 5.x
- TypeScript 7.0.2 是较新的版本，可能存在 breaking changes

**解决方案**：

**方案 1：降级 TypeScript**
```json
{
  "devDependencies": {
    "typescript": "^5.3.3"
  }
}
```

**方案 2：升级 @typescript-eslint**
```json
{
  "devDependencies": {
    "@typescript-eslint/eslint-plugin": "^6.0.0",
    "@typescript-eslint/parser": "^6.0.0"
  }
}
```

**方案 3：锁定版本组合（推荐）**
```json
{
  "devDependencies": {
    "typescript": "^5.0.0",
    "@typescript-eslint/eslint-plugin": "^5.62.0",
    "@typescript-eslint/parser": "^5.62.0"
  }
}
```

**建议**：优先选择方案 3，确保版本兼容性
