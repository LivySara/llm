## AIO-Forward 对 Origin / Referer 做了来源校验

很多公司为了防止以下安全风险：

- **CSRF（跨站请求伪造）**：攻击者诱导用户在已登录状态下执行非预期操作
- **恶意跨域调用**：未授权的第三方站点访问敏感资源
- **内网接口滥用**：防止内部 API 被外部页面非法调用

## SSH: DEVCLOUD 配置过程

### 1. 生成 SSH 密钥
```bash
ssh-keygen -t rsa -b 4096
```

### 2. 配置 SSH config
编辑 `~/.ssh/config`：
```
Host devcloud
    HostName <服务器IP>
    Port 36000
    User root
    IdentityFile ~/.ssh/id_rsa
```

### 3. 添加公钥到服务器
```bash
ssh-copy-id -p 36000 user@server_ip
```

### 4. 测试连接
```bash
ssh devcloud
```

## 表单布局问题

### 问题描述
表单 label 宽度会动态变化，表单项的显隐影响其他表单项 label 宽度变化

### 解决方案

**方案 1：使用 CSS visibility**
```css
.form-item {
  visibility: hidden;
}
```

**方案 2：使用 Vue 的 v-show**
```vue
<div v-show="condition">...</div>
```

**方案对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| visibility: hidden | 保持布局、不重新渲染 | 元素仍在 DOM 中 |
| v-show | 保持布局、切换快 | 元素仍在 DOM 中 |
| v-if | 完全销毁元素 | 频繁切换性能差 |

**推荐**：频繁切换使用 `v-show`，很少切换使用 `v-if`

### 补充说明

**疑问**：用 `v-show` 依然不能全部解决某些表单项 label 宽度固定的情况

**问题场景**：
- 某些表单项的 label 宽度是**动态变化**的
- 当这些表单项显隐切换时，会影响其他固定宽度 label 的对齐

**根本原因**：
- 动态 label 和固定 label 混用导致布局不一致
- 显隐切换时 DOM 结构变化影响布局计算

**解决方案**：
```css
.form-label {
  width: 120px; /* 固定宽度 */
  flex-shrink: 0; /* 防止收缩 */
}
```

~~或使用 `text-overflow: ellipsis` 处理长文本~~ ❌ **此方案不可行**

**实际问题**：
- 已给组件传了 `label-width` 属性，但未生效
- 说明组件内部可能没有正确处理这个 prop，或者 CSS 优先级问题

**排查方向**：
1. 检查组件是否正确接收 `label-width` prop
2. 检查 CSS 选择器优先级是否被覆盖
3. 查看组件文档确认正确的属性名（可能是 `labelWidth` 或其他）

**原因分析**：
- 组件可能使用 CSS 变量或内联样式，覆盖了传入的宽度值
- 或者属性名大小写不匹配
